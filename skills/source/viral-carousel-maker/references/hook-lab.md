# Hook Lab

The Hook Lab runs before template selection. Do not choose a template until the hook has enough tension to justify the carousel.

## Required output

Generate at least 5 hook candidates:

- 2 safe but strong candidates
- 2 bolder candidates
- 1 uncomfortable candidate that may be too sharp

For each candidate, record:

- `hook`
- `archetype`
- `old_belief`
- `new_belief`
- `risk`
- `proof_needed`
- `why_it_stops_scroll`

## Banned default openers

Do not use these unless the user explicitly requests a search/tutorial angle:

- "How to..."
- "Here are..."
- "Here's how..."
- "Want to...?"
- "Save this..."
- "Follow for..."
- "Comment below..."
- "The ultimate guide..."
- Plain numbered "tips/ways/steps" hooks

## Hook archetypes

- Shock stat: lead with a number that violates expectation.
- "X is a lie": name the false belief directly.
- I got this wrong: earned confession from experience.
- Authority confession: credible person admits the old model failed.
- Private-room observation: feels like overhearing something true.
- Unpopular opinion: polarizing but defensible.
- Data contradiction: familiar advice failed in the evidence.
- Enemy belief: name the bad advice, habit, tool, or system.
- Proof-first receipt: show result, then explain.
- Identity mirror: makes the target viewer feel recognized.

## Scoring rubric

Score each hook from 1-10:

- Tweet test: would it work as a standalone Threads post?
- Contradiction: does it challenge an existing belief?
- Specificity: does it avoid vague "content/growth/value" sludge?
- Proof: can the user support it?
- Identity tension: would the right viewer feel seen or called out?
- Curiosity: does it make the next slide necessary?
- Risk: is it bold without becoming false or cruel?

Select the hook only when:

- It creates a clear old-belief-to-new-belief shift.
- It can be supported by the carousel body.
- It is short enough to read instantly on mobile.
- It does not require fake urgency.

If no candidate clears `8.5/10`, generate a new batch before drafting.
